## Convenções de Commits

Adotamos o padrão `<tipo>(escopo opcional): descrição breve` para manter o histórico claro e consistente.  
Principais tipos: **feat** (nova funcionalidade), **fix** (correção de bug), **docs** (documentação), **style** (formatação), **refactor** (refatoração), **test** (testes), **chore** (manutenção).  
Exemplo: `feat(auth): adicionar login com OAuth`
